# const/final은 정말 상수일까?

TypeScript를 공부하다가 이런 코드를 적었습니다.

<!--more-->

```ts
const obj = {
    name: 'kang'
};

obj.name = 'lim';

console.log(obj);
```

처음에는 `const`인데 왜 값이 바뀌지? 라는 생각이 들었습니다.

그냥 자연스럽게 `const`를 "값이 안 바뀌는 상수" 정도로 생각하고 있었던 것 같습니다.

그런데 생각해보니 Java의 `final`도 비슷합니다.

TypeScript를 보다가 Java 개념을 다시 보게 됐습니다.

## primitive 값은 수정되지 않는다.

먼저 primitive 값부터 보면

### Java

```java
final int number = 10;
number = 20; // compile error
```

### TypeScript

```ts
const number = 10;
number = 20; // error
```

두 언어 모두 값 변경이 불가능합니다.

그래서 자연스럽게 `const`, `final`을 상수처럼 받아들이게 됩니다.

<!--more-->

## 그런데 객체는 다르다.

### Java

```java
class Person {
    String name;
}

final Person person = new Person();
person.name = "kim"; // 가능
```

### TypeScript

```ts
const obj = {
    name: 'kim'
};

obj.name = 'lee'; // 가능
```

분명 `final`, `const`인데 객체 내부 값은 수정이 가능합니다.

왜 이런 동작을 하는 걸까?

## const/final은 참조를 고정한다.

핵심은 객체 자체가 아니라 "참조(reference)"를 고정합니다.

```text
변수 -----> 객체
```

`const`, `final`은 변수와 객체를 연결하는 참조를 다른 객체로 바꾸지 못하게 막습니다.

그래서 아래 코드는 불가능합니다.

### Java

```java
person = new Person(); // error
```

### TypeScript

```ts
obj = {
    name: 'park'
}; // error
```

하지만 객체 내부 상태를 수정하는 것은 가능합니다.

## 진짜 immutable 객체를 만들고 싶다면?

`const`, `final`만으로는 객체를 완전히 immutable하게 만들 수 없습니다.

### Java

보통 아래와 같은 방식으로 불변 객체를 만듭니다.

- 필드를 `private final`로 선언
- setter 제거
- 생성자로만 값 초기화
- 필요하면 `record` 사용

### TypeScript

TypeScript에서는 아래와 같은 방법을 사용합니다.

```ts
const obj = {
    name: 'kim'
} as const;
```

또는:

```ts
Object.freeze(obj);
```

`as const`는 readonly + literal type 효과를 줍니다.

## 마무리

처음에는 `const`와 `final`을 그냥 상수라고만 생각하고있었는데
다시 보니 둘 다 "참조 재할당을 막는다"는 의미였습니다.

Java에서 DTO 클래스를 만들 때도 `private final`을 많이 사용하는데
정작 이런 기본적인 동작을 헷갈렸습니다..

기초가 중요합니다..
